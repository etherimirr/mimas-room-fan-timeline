# Codex 项目总提示词

请构建一个互动叙事网页产品，而不是普通博客或普通游戏官网。

这个项目基于《未麻的部屋》的世界观与叙事气质。用户以“粉丝”身份进入一个偶像的个人主页 / 日记站。网站表面上看起来温柔、可爱、日常、真实，像一个 90 年代的个人主页；但随着时间推进和用户不断返回，网站会逐渐出现细微异常，让用户慢慢感觉这个网站记住了自己、在回应自己、甚至在观察自己。

## 产品定义

核心体验：
1. 用户以“粉丝”身份进入，而不是以旁观者身份浏览。
2. 网站会随着用户首次进入后的时间推进而变化。
3. 不同用户因为进入时间不同，会拥有不同的解锁节奏。
4. 系统必须记住用户行为、阅读进度、分支和周目。
5. 网站应支持后续持续更新日记、事件、分支和功能。

一句话概括：
这是一个时间驱动、可连载、可分支、可周目的互动叙事网页产品。

## 剧情目标

- 表面上这是一个温柔、可爱、日常的个人主页 / 日记站。
- 实际上它会逐渐让用户感觉不对劲。
- 用户会慢慢觉得：这个网站记住了我、在回应我、甚至在观察我。
- 整体是沉浸式第一人称粉丝体验。
- 重点不是 jump scare，而是慢慢失真、现实和虚构混淆的心理感。

非常重要：
不要直接做成恐怖游戏 UI，要先正常、纯真、日常，再逐步异常。

## 风格要求

### 视觉风格
- 90 年代早期个人主页 / 日记网页感
- old web / 简单 HTML 感
- 颜色以浅粉、白、柔和蓝为主
- 看起来质朴、真实、略笨拙，不要太现代 UI
- 不要做成 SaaS dashboard，也不要做成高科技赛博面板

### 情绪风格
- 表面温柔、清新、可爱
- 底层有不安、窥视感、身份错位感
- 重点是“越看越不对劲”

### 交互风格
- 交互不能太游戏化
- 尽量像真的在浏览一个个人主页
- 异常效果要克制，少量但有效
- glitch、文本变化、延迟更新、隐藏句子比大面积特效更重要

### 文案风格
- 语言一开始应该自然、日常、像偶像博客
- 后面才逐渐出现微妙异常
- 不要从第一屏就写得像 horror game

## 系统规则

1. 每个用户必须有独立时间线。
2. 所有主要事件基于用户首次进入时间 `first_seen_at` 解锁。
3. 用户状态必须记录：
   - `first_seen_at`
   - `last_seen_at`
   - `flags`
   - `read_entries`
   - `loop`
4. 内容必须数据驱动，不能写死在前端组件里。
5. 必须支持后续持续新增日记、事件、分支、周目内容。
6. 前期可用 `localStorage`，后期要可迁移到数据库。
7. 代码结构要可维护，不要把剧情逻辑散落在组件里。

## 工程原则

- 时间推进必须围绕 `elapsed_time = now - first_seen_at`。
- 主要剧情内容应支持 `unlock_after_hours` 这类相对时间解锁。
- 氛围内容应支持 `only_at_night` 这类时间条件。
- 系统应为事件驱动，不要写死成单一路线流程。
- 事件来源可以包括留言、点击、页面访问、重复访问、夜间访问和时间到达。
- 分支必须真实存在，而不是只做表面文案差异。
- 周目系统必须可扩展，用于未来的 New Game+ 内容。

## 内容组织

所有内容应通过 JSON 或类似数据文件管理，例如：
- 日记 entries
- 事件 events
- 条件 rules
- 用户状态 state

不要把大量剧情文本直接硬编码进 React 组件。

## 当前任务范围

这次先不要做完整产品，只做 MVP：
- 首页
- 日记列表 / 日记详情
- 用户首次进入时间记录
- 24 小时后解锁第二篇日记
- 至少一个 based on flag 的分支
- 一个克制的异常效果
- 为未来扩展事件、周目和更多内容预留空间

## 最终目标

请优先构建一个“先正常、后异常”的互动叙事系统，而不是一个单纯的复古风页面。
